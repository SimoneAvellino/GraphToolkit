# Graph CSV Format

This project models a graph using multiple CSV files. Each file represents one label (node or edge).

## File naming

-   The filename is the label for all records inside that CSV (e.g., `Person.csv`, `KNOWS.csv`).

## Node files

-   Structure: one column named `id:ID(<EntityLabel>)`.
-   Each row holds a single node identifier for that label.
-   Example (`Person.csv`):

```
id:ID(Person)
123
456
```

## Edge files

-   Structure: one column named `:START_ID(<EntityALabel>)|:END_ID(<EntityBLabel>)`.
-   Each row holds a pipe-delimited pair `A|B` where `A` is the start node id and `B` is the end node id.
-   Example (`KNOWS.csv`, edges from `Person` to `Person`):

```
:START_ID(Person)|:END_ID(Person)
123|456
456|789
```

## Notes

-   Labels in column headers must match the corresponding node labels.
-   IDs must match those declared in the node CSVs.
